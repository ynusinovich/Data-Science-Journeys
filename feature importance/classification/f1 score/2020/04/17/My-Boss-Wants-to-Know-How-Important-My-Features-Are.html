Theory

What is feature importance?

Feature importance is the relative weights of your features in X in predicting your target variable y.

For this post, I will focus on the feature importances that are included in your models in Scikit-learn, a machine learning library in Python.

You can also derive feature importance by (1) shuffling your data for a feature and seeing how that affects your predictions, (2) removing a feature, retraining your model, and seeing how that affects your predictions, and (3) fit local linear models around small changes in the features to explain specific target variable results. You can read more about those techniques at this post on Towards Data Science. These methods are more accurate but more computationally intensive.

Examples

Why do we care about feature importance?

  • A client wants you to make inferences about which feature affects home prices most.
  • You want to know whether your model implicitly says that people of a certain race should not see advertisements for your company's retirement plan.
  • Your supervisor wants you to figure out the groups of words most indicative of a spam email.

Code

How to extract and graph the feature importances included in Scikit-learn.

For this example, we will be using a dataset about retirement plans. Our target variable will be eligibility for the 401(k) plan.

  • e401k = 1 if eligible for 401(k)
  • inc = annual income (\$)
  • marr = 1 if married
  • male = 1 if male
  • age = age (years)
  • fsize = family size
  • nettfa = net total financial assets (\$1000s)
  • p401k = 1 if participates in 401(k)
  • pira = 1 if has an IRA
  • incsq = inc2
  • agesq = age2

We will drop the feature that indicates whether or not the potential customer is already participating in a 401(k) plan. That wouldn't be very interesting to model since that offers no insight on features affecting eligibility. Grab the dataset from Google Drive if you want to follow along.

Imports, Define Variables, and Train, Test, Split

  • Start by importing the libraries and loading the data we will need:
import numpy as np
import pandas as pd

from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler, MinMaxScaler

from sklearn.linear_model import LogisticRegression
from sklearn.naive_bayes import MultinomialNB
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import BaggingClassifier, RandomForestClassifier, ExtraTreesClassifier, AdaBoostClassifier

from sklearn.metrics import f1_score

import matplotlib.pyplot as plt
import matplotlib.style as style
import seaborn as sns

style.use('fivethirtyeight')

df = pd.read_csv("401ksubs.csv", sep = ",")

  • Define the features and the target variable (remember, we're not using p401k):
X = df[["inc", "marr", "male", "age", "fsize", "nettfa", "pira", "incsq", "agesq"]]
y = df["e401k"]

X_train, X_test, y_train, y_test = train_test_split(X,
                                                    y,
                                                    test_size = 0.33,
                                                    stratify = y,
                                                    random_state = 42)
  • I've set a random_state of 42 so that you can follow along and get the same results. If you're not trying to get the same results, you can leave the random_state blank.
  • I am only including only one value of each hyperparameter here, since hyperparameter tuning is not the focus of this blog post. Feel free to play around with the hyperparameters yourself!

Logistic Regression Most Important Features 

sc = StandardScaler()
X_train_sc = sc.fit_transform(X_train)
X_test_sc = sc.transform(X_test)
lr = LogisticRegression(C = 2.32995, penalty = "l2", solver = "liblinear", random_state = 42)
lr.fit(X_train_sc, y_train)
print("Accuracy for training data: ", lr.score(X_train_sc, y_train))
print("Accuracy for test data: ", lr.score(X_test_sc, y_test))
print("F1 for training data: ", f1_score(y_train, lr.predict(X_train_sc)))
print("F1 for test data: ", f1_score(y_test, lr.predict(X_test_sc)))         

feature_importances = lr.coef_.reshape(len(X_train.columns),1)
feature_importances_df = pd.DataFrame(data = feature_importances,
                                      index = X_train.columns,
                                      columns = ["feature_importances"])

# Plot Feature Importances
plt.bar(x = feature_importances_df.index,
        height = feature_importances_df["feature_importances"])
ax = plt.gca()
fig = plt.gcf()
fig.set_size_inches(11,8)
ax.set_ylabel("Beta Coefficient for Feature", size = 18)
ax.set_xlabel("Feature", size = 18)
ax.set_title("Effect of Each Feature on 401(k) Eligibility", size = 25)
plt.show()

  • The feature importances are the beta coefficients of the logistic regression. For more information about beta coefficients, see this post on the Display R Blog.
  • The most significant features are inc, agesq, and age, in that order.

Bootstrap Aggregated (Bagged) Decision Tree Most Important Features 

sc = StandardScaler()
X_train_sc = sc.fit_transform(X_train)
X_test_sc = sc.transform(X_test)
bag = BaggingClassifier(max_samples = 0.3,
                        n_estimators = 15,
                        random_state = 42, 
                        base_estimator = DecisionTreeClassifier(max_depth = 4, 
                                                                min_samples_leaf = 3,
                                                                min_samples_split = 11,
                                                                random_state = 42))
bag.fit(X_train_sc, y_train)
print("Accuracy for training data: ", bag.score(X_train_sc, y_train))
print("Accuracy for test data: ", bag.score(X_test_sc, y_test))
print("F1 for training data: ", f1_score(y_train, bag.predict(X_train_sc)))
print("F1 for test data: ", f1_score(y_test, bag.predict(X_test_sc)))         

feature_importances = np.mean([tree.feature_importances_ for tree in bag.estimators_], axis = 0)
feature_importances_df = pd.DataFrame(data = feature_importances,
                                      index = X_train.columns,
                                      columns = ["feature_importances"])

# Plot Feature Importances
plt.bar(x = feature_importances_df.index,
        height = feature_importances_df["feature_importances"])
ax = plt.gca()
fig = plt.gcf()
fig.set_size_inches(11,8)
ax.set_ylabel("Gini Feature Importance", size = 18)
ax.set_xlabel("Feature", size = 18)
ax.set_title("Effect of Each Feature on 401(k) Eligibility", size = 25)
plt.show()

Accuracy for training data:  0.6971355004827808
Accuracy for test data:  0.6863770009800719
F1 for training data:  0.5669581224114129
F1 for test data:  0.5522388059701493
  • The importance of a feature is computed as (normalized) total reduction of the criterion brought by that feature, also known as the Gini importance, for the average decision tree.
  • If you've also set the max_features parameter of your BaggingClassifier to anything less than 1, you won't be able to use this code to average the features on the decision trees and compare their importances.
  • The most significant features are the nettfa, incsq, and inc, in that order.

Conclusions from the Code

  • My choice of model for this example would be AdaBoost. It has a very high accuracy (correct predictions out of total predictions) and the highest F1 score (avoids false positives and false negatives) of any of the models.
  • (1) Did you want to run a k-Nearest Neighbors Classifier, a Support Vector Classifier, or a Gaussian Naive Bayes classifier for this example? (2) Did you want more precise measures of feature importance than this introductory example provides?
  • In that case, you can remove each feature (drop-column feature importance) or shuffle each feature (permutation feature importance), and then see how the lack of that feature affects your accuracy and F1 score.
  • You can also use more advanced model interpretability packages like LIME or SHAP.
